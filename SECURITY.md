# Security Policy

## Supported Status

This repository is under active development. Security hardening is a core project priority, especially around device identity, OTA delivery, and AI-triggered control paths.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Instead, report security concerns privately to the project maintainer through a dedicated security contact once configured for this repository.

Until that is set up, keep disclosures private and include:

- affected component
- impact summary
- reproduction details
- suggested mitigation if known

## Security Principles

- unique device credentials
- encrypted transport
- signed firmware artifacts
- least-privilege operator access
- audited remote command execution
- explicit validation for AI-generated actions
