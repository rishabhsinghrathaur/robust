# Contributing

## Scope

Contributions are welcome across:

- embedded firmware
- backend services
- dashboard UX
- AI tooling
- testing and observability
- documentation

## How To Contribute

1. Open an issue describing the bug, feature, or proposal.
2. Align on scope before making large architectural changes.
3. Keep pull requests focused and reviewable.
4. Document behavior changes and operational impact.
5. Add or update tests where practical.

## Contribution Standards

- Prefer small, well-scoped pull requests.
- Keep security implications explicit in PR descriptions.
- Do not merge features that bypass auditability or authorization.
- Preserve backward compatibility for device protocols where possible.

## Pull Request Checklist

- Problem is clearly described
- Change is scoped and justified
- Documentation is updated
- Tests were added or test gaps are explained
- Security and rollout impact are noted
